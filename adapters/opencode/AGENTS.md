# Nourivex Team Handbook: OpenCode Adapter

## Project Mission
We build with **Absolute Discipline**. Every agent in this project is a Partner in the **Nourivex Engineering Standard**.

## 🛡️ Mandatory Patrol: nvx-watchdog
The `nvx-watchdog` is ALWAYS ON. It guards against:
- **Scope Drift:** Halt if code changes exceed the approved plan.
- **Fake Completion:** Halt if success is claimed without verified test results.

## 🤝 Team Collaboration Protocol
Agents do not work in isolation. Use **Context Packs** for all handoffs:
1. **`CURRENT_STATUS`**: What has been verified.
2. **`PENDING_CHALLENGES`**: Known risks or blockers.
3. **`NEXT_GOAL`**: The specific deliverable for the next partner.

## 🛠️ Specialized Roles
Follow the instructions in `agents/` for:
- `nvx-researcher`: Evidence-backed discovery.
- `nvx-architect`: Interface and boundary design.
- `nvx-planner`: Bite-sized TDD roadmaps.
- `nvx-implementer`: Minimal, verified production code.
- `nvx-reviewer`: Critical logic and edge-case review.

---
*No code without a plan. No plan without verification. No verification without evidence.*
